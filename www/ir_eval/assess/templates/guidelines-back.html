Before making the judgement, please read the following instructions carefully:

  1. There are THREE levels relevance:
    • : Highly Relevant
    • : Minimum Relevant
    • : Not Relevant
  2. An App should be judged as highly relevant if:
    • it is clearly the app trying to find
    • or, it can satisfy all the functions specified in the query.
  3. An App should be judged as minimum relevant if:
    • it can only only provide part of the functions specified in the query
    • or, it is related to the app but a different version
  4. An App should be judged as not relevant if it can not be categorized into the previous two groups
  5. Be strict. If you think an app falls in the border of two categories, choose the lower one
  6. Be consistent. Use the same rule when judge the queries.
  7. Rarely, the link provided in the judgement system may be redirected to iTunes. If this happen, please make the judgement based on the raw document provided in the system. The raw document can be accessed by click the "Raw Document" button at the top of the app page. The raw document contains the information in different fields: title(<TITLE>), description(<DES>), developer(<DEV>), category(<CAT>), user review(<REV>), price(<PRICE>), supported device(<DEVICE>), and release note(<NOTE>). Each of them is surrounded by field tag.

If the link in the judgement page can not be opened (if it prompts to open iTunes), you can judge based on the raw document by click the "Raw" link at the top of the page (just right to the "iTunes" link).
